How It Started

At 17, I stepped into engineering, not just to study, but to explore. I was curious, often beyond the textbook.

I experimented.
I built things.
I don't wait to be told how - I figure it out, build it, and improve it.

I explored how web authentication flows work by recreating simple login interfaces. This helped me understand how user input is handled and where potential vulnerabilities can arise, which sparked an early interest in application security and the importance of designing systems responsibly.

I installed Ubuntu alongside Windows to explore a different environment and learn by doing. I didn't always start with theory - I focused on outcomes, then worked backwards to understand the fundamentals.

That phase shaped how I work today.
Curiosity over perfection. Learning by doing.
